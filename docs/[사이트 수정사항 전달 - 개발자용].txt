[사이트 수정사항 전달 - 개발자용]

0) 목적/핵심 (가장 중요)
- 초보/중급 유저는 "지금 당장 배팅 가능한(현재 시간과 가까운)" 수익경기를 선호함
- 2~3일 뒤 경기보다 "가까운 시간대 경기"를 우선 노출해야 함
- 따라서 정렬/필터 UX는 아래 우선순위를 기본으로 제공해야 함

기본 정렬(추천 기본값):
A안) "현재시간과 가장 가까운 경기" 우선 정렬 (kickoff_time ASC)
B안) "가장 가까운 경기" 범위 내에서 수익률 높은 순 (kickoff_time ASC → profit_pct DESC)

※ 구현 팁:
- '가까운 경기' 범위(예: 0~6시간/0~12시간/오늘경기) 프리셋 제공하면 더 직관적임


1) 좌측 메뉴: 벨루벳/슈어벳 수익경기 카운트 표기
- 좌측 네비게이션에 다음 형식으로 표시:
  - 벨루벳 (3)
  - 슈어벳 (5)
- 숫자는 "현재 선택된 필터/정렬 조건 적용 후, 화면에 노출 가능한 수익경기 개수" 기준
  (또는 전체 수익경기 개수 기준으로 고정할지 옵션화 가능)

요구사항:
- 실시간 데이터 갱신 시 카운트도 즉시 업데이트
- 0일 경우: 벨루벳 (0), 슈어벳 (0) 표기 유지


2) 사이트 선택(체크박스) 기능 추가
- 사용자가 비교/표시할 북메이커를 선택할 수 있어야 함
예시 UI:
- [체크] Pinnacle
- [미체크] BookmakerXYZ
- [미체크] BookmakerABC ...

요구사항:
- 최소 1개 사이트는 반드시 선택되어야 함 (전부 해제 방지)
- 체크/해제 시 경기 리스트 및 카운트 즉시 재계산/갱신
- 선택된 사이트 조합에 따라 매칭 가능한 경기만 보여주기


3) 정렬 기준 선택 메뉴 추가 (좌측 대시보드)
- 유저가 다음 보기 방식을 선택할 수 있어야 함:
  1) 수익률 기준으로 보기 (profit_pct DESC)
  2) 최근 가장 빠른 시간대(가까운 경기) 기준으로 보기 (kickoff_time ASC)

추가 요구:
- "가까운 경기 + 수익률 높은 순" 복합 정렬 옵션도 반드시 제공
  (kickoff_time ASC → profit_pct DESC)
- 기본값은 "가까운 경기 + 수익률 높은 순" 추천


4) 수익률 표기 소수점 자리 제한
- 수익률 표시 예: 1.56%
- 소수점은 2자리까지만 표시(= 총 2 decimal)
- 반올림 적용 (예: 1.567% → 1.57%)

주의:
- 화면 표시만 제한, 내부 계산은 기존 정밀도 유지


5) 신규 수익경기 등록 시 알림음 + 음소거 옵션
- 새로운 수익경기가 등록(리스트에 새로 등장)되면 "딩동" 같은 알림음 재생
- 사용자가 음소거(뮤트) 토글 가능해야 함
- 브라우저 정책상 자동재생 제한이 있을 수 있으니 첫 사용자 인터랙션 후 활성화 방식 고려

요구사항:
- 알림 트리거 기준 정의:
  - "기존에 없던 경기 ID가 새로 유효 리스트에 들어온 경우" 또는
  - "기존 경기였으나 profit 조건 충족으로 수익경기로 전환된 경우"
- 알림 빈도 제한(스팸 방지):
  - 같은 경기 재등장 시 일정 시간(예: 30~60초) 내 중복 알림 방지


6) (중요) 리스트 정렬 동작 정의 (최종)
- 유저가 원하는 핵심 UX:
  - '현재시간과 가장 가까운 수익경기'가 최상단에 보여야 함
  - 그리고 가까운 경기들 중에서는 수익률 높은 순으로 정렬되면 좋음

권장 최종 정렬 옵션 3개:
(1) 가까운 경기 우선: kickoff_time ASC
(2) 수익률 우선: profit_pct DESC
(3) 가까운 경기 + 수익률: kickoff_time ASC → profit_pct DESC  [기본값 추천]

추가 옵션(있으면 더 좋음):
- "오늘 경기만" 토글
- "N시간 이내(예: 6시간/12시간)" 프리셋 필터
- 경기 시작 시간이 지난 경기 자동 숨김
